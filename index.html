WHY IS ES6 SUCH A BIG DEAL?

ECMAScript 5th Edition

  • Scaled back subset of abandoned 4th Edition
  • Focused mainly on bug fixes and compatibility

ESHarmony

Stuff in ES4 that didn't make it into ES5

ES.next

Subset of features from ESHarmony ready
for next release (i.e. ES6)

ECMAScript, 6th Edition

  • Approved in June 2015
  • Officially renamed ECMAScript 2015

(ES5 approved in 2009, ES3 approvied in 1999)

"Subset"

Most significant update to the core of the language since the introduction of [] and {} in JavaScript 1.2 (approved in 1997)!

Reduced friction

Incorporates concepts common amongst other popular languages with the goal of making it easier to get up and running for developers used to those languages

Using es6 today

limited Browser availability

transpilation tools

  • babel (we use this)
  • traceur compiler
  • TypeKit
  • nodejs implements some features, and some via flags

...Rest parameters

function sum() {
  var value = 0;
  for (var i = 0; i < arguments.length; i++) {
    value += arguments[i];
  }
  return value;
}
function brokenSum() {
  return arguments.reduce(function (x, y) {
    return x + y;
  });
}
function fixedSum(...values) {
  return values.reduce(function (x, y) {
    return x + y;
  });
}
When you want to deal with arbitrary arguments, you aren't always garunteed an array. Using rest parameters, you are garunteed an array.

New core prototypes

Set

Collection of unique values

var set = new Set();

set.add('123');
set.add(123);

set.has('123');   // true
set.has(123);     // true

set.size;         // 2

set.delete(123);
set.clear();

Starting values

var startingValues = [
  'abc',
  'xyz',
  123,
  456,
  [],
  [],
  true,
  false,
  null
];
var set = new Set(startingValues);

console.log(set.size);   // 9 (the empty arrays are unique objects)

Iteration over a set

for (let item of set) {
  console.log(item);
}

Map

Collection of unique keys paired with values

var map = new Map();
var foo = 1;
var bar = 'a';

map.set(foo, bar);

map.get(foo);   // bar

map.has(foo);   // true

map.size;       // 1

map.delete(foo);
map.clear();

Starting values

var startingValues = [
  [1, 'a'],
  [[], false]
];
var map = new Map(startingValues);

Iteration over a map

for (let [key, value] of map) {
  console.log(`${key}: ${value}`);
}
